# 0001: Scaffold the Astro + Vercel project foundation

## Story

As a developer, I want a deployable Astro + TypeScript project wired to Vercel, so that every later feature has a running, continuously-deployed baseline to build on.

## Acceptance Criteria

- A push to the main branch produces a successful Vercel deployment reachable at a live URL
- The project builds locally with a single documented command and reports zero TypeScript errors
- A placeholder home route renders and returns HTTP 200 in production
- Server-rendered (SSR) routes are supported, confirmed by at least one route that renders per-request
- README documents local dev, build, and deploy steps

## Technical Notes

- Greenfield — no existing patterns; all files below are created new
- Well-trodden path (`astro add vercel`, strict `tsconfig`, one SSR route); main risk is the external Vercel deploy pipeline + live URL, but standard
- Hard prerequisite for every other story in the epic
- Resolve early (affects later stories): serverless Postgres provider (Vercel Postgres vs Neon)
- Affected files (create): `package.json`, `astro.config.mjs`, `tsconfig.json`, `src/pages/index.astro`, one SSR route (e.g. `src/pages/api/health.ts`), optional `vercel.json`, update `README.md`
