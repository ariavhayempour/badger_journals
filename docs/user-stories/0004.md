# 0004: Optimize and self-host all imagery

## Story

As an end user, I want images to load quickly and reliably, so that pages render fast without depending on third-party image hosts.

## Acceptance Criteria

- No image is delivered as an inlined base64 data URI
- The campus photo and all other images are served from the project's own hosting (no imgur or external image links)
- Images are served in an optimized, appropriately sized format for the viewport
- Every content image has descriptive alt text
- Static content pages meet Core Web Vitals thresholds (LCP, CLS, INP) on a mid-tier mobile profile

## Technical Notes

- Astro `<Image>` handles resizing/format optimization; mechanical work is pulling external assets (imgur), stripping ~270KB of base64, and adding alt text
- Core Web Vitals bar (LCP/CLS/INP on mid-tier mobile) is a measurable acceptance gate that may require iteration
- Assumption: image count is modest; a large asset inventory would push toward 5 points
- Affected files (create/edit): `src/assets/images/*`, `astro.config.mjs` (image config), image references across page components
