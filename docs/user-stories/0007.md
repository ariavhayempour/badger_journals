# 0007: Render the meetings calendar from repo data

## Story

As an end user, I want to see upcoming club meetings on a calendar, so that I know when and where meetings happen.

## Acceptance Criteria

- The meetings page renders meeting dates sourced from developer-edited repo data files
- Each meeting displays its date and key details (e.g., title/topic, time, location as present in the data)
- Adding or editing a meeting in the repo data updates the rendered calendar after deploy
- The calendar is keyboard-navigable and readable on mobile widths
- An empty state displays when no upcoming meetings exist

## Technical Notes

- Typed data files → render; no DB. The old site's calendar had no real meeting data wired in (meetingDates empty) — this replaces the hand-rolled JS calendar
- Main effort is an accessible, keyboard-navigable, mobile-readable calendar UI + empty state
- Assumption to confirm: "calendar" can be a chronological list rather than a full interactive month-grid widget — a grid widget pushes this to 5 points
- Affected files (create): `src/content/meetings/*.{md,ts}`, `src/pages/meetings.astro`, `src/components/MeetingsCalendar.tsx` (React island)
