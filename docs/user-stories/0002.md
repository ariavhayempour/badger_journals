# 0002: Migrate content to componentized pages with real per-page URLs

## Story

As an end user, I want each section and article to live at its own shareable URL, so that I can link to, bookmark, and find specific content directly instead of navigating a single-page toggle.

## Acceptance Criteria

- Each of these has its own distinct, directly-loadable URL: Hub landing, each of the 3 digests (Cardiovascular, Cancer, Neuroscience), each review page, Mission, Team, Reviews, Meetings, "Create the Next Digest", and Contact
- Loading any page URL directly (not via in-site navigation) renders that page's full content
- Each page emits its own title and Open Graph/meta tags reflecting that page's content
- All legacy content from the old index.html is present on the corresponding new page with no content lost
- Navigating between pages updates the browser URL and supports back/forward buttons

## Technical Notes

- Greenfield — establishes the first shared layout, header/footer, and SEO/meta patterns (first-time tax); depends on 0001
- ~11 named routes; Astro file-based routing and back/forward come mostly free — real cost is content-migration fidelity ("all legacy content preserved")
- Assumption: legacy content volume / number of review pages is bounded and available; unknown review-page count is the main uncertainty
- Affected files (create): `src/layouts/BaseLayout.astro`, `src/components/{Header,Footer,SEO}.astro`, `src/pages/index.astro`, `src/pages/digests/{cardiovascular,cancer,neuroscience}.astro`, `src/pages/reviews/[slug].astro`, `src/pages/{mission,team,reviews,meetings,create-next-digest,contact}.astro`, `src/content/` data
