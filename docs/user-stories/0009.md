# 0009: Submit inquiries, join requests, and "start a digest" through one unified flow

## Story

As an end user, I want to send the club an inquiry, a join request, or a "start the next digest" proposal, so that I can reach the team without a third-party form service.

## Acceptance Criteria

- The Contact, join, and "Create the Next Digest" actions each submit through the site's own backend (no Formspree)
- Each submission captures the submitter's name, wisc.edu email, submission type, and message, and persists it
- Submitting with a missing required field or a non-wisc.edu email shows a specific inline error and does not persist
- A successful submission shows a confirmation and clears the form
- All three legacy Formspree forms are removed from the site

## Technical Notes

- Depends on 0006; reuses 0008's submission pattern (cheaper if 0008 lands first) but adds submission-type discrimination + 3 UI touchpoints
- Consolidates the three legacy forms that POST to one Formspree endpoint (f/mqeoeogq) into the DB-backed flow so all submissions land in one admin dashboard
- "All legacy Formspree removed" is a verifiable cleanup criterion
- Affected files (create): `src/pages/api/inquiry.ts`, `src/components/InquiryForm.tsx`, `src/db/inquiry.ts`; edit `contact.astro`, `team.astro`/join, `create-next-digest.astro`
